## О проекте

Американская B2B-платформа для школ коммерческого вождения и мобильного приложения, которое помогает будущим дальнобойщикам подготовиться к экзамену на CDL (Commercial Driver's License). Я пришёл в стартап на раннем этапе и прошёл путь от телеграм-ботов до полноценной микросервисной инфраструктуры для трёх взаимосвязанных продуктов.

## Что сделал

- Спроектировал и реализовал микросервисную архитектуру серверной части на FastAPI/Flask с PostgreSQL и Redis, заменив монолит
- Настроил VPC и API Gateway для безопасной коммуникации между сервисами и обеспечил единую систему авторизации — пользователь регистрируется в одном продукте и входит во все остальные
- Разработал REST API для управления пользователями, учебным контентом и бизнес-логикой CRM
- Интегрировал сторонние сервисы: Auth0, Firebase, Supabase, Strapi
- Реализовал мобильные фичи на Flutter: учебные модули и отслеживание прогресса студентов
- Создал специализированный редактор контента для локализации, упрощающий перевод больших объёмов текста
- Развернул мультиязычный сайт на Strapi с i18n для глобальной аудитории
- Автоматизировал лидогенерацию через телеграм-боты, анализирующие тематические группы

## Сложности

Работал в одиночку над всем стеком: бэкенд, мобилка, вёрстка сайтов, DevOps. Главный вызов — перейти с монолита на микросервисы без потери связности между тремя продуктами. Решение: единая сеть с общим пользователем через API Gateway и Auth0. Система обслуживает более 4 000 активных пользователей в месяц.
